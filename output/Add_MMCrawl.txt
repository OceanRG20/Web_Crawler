====================================================
ADD_MMCrawl — DATA CLEANING & NORMALIZATION RULES
(applied INSIDE this step, before writing JSON)
====================================================

When cleaning or transforming the THREE metadata fields  
"Square footage (facility)", "Number of employees", and "Estimated Revenues",  
you MUST apply the following strict rules:

------------------------------------------------------------
1. SQUARE FOOTAGE (FACILITY)
------------------------------------------------------------
• Work on the value AFTER it is copied from Raw_Data.  
• If the field is "refer to site" or genuinely empty → leave as-is (do not change).

• For each numeric square-foot value (treat semicolon-separated parts independently):

  - Remove commas before interpreting the number.  
  - If absolute value < 1,000,000 → convert to **"~Xk"** or "**~Xk ft²**":
      • 26,000 → "~26k"  
      • 51,000 ft² → "~51k ft²"  
  - If absolute value ≥ 1,000,000 → convert to **"~XM"** or "**~XM ft²**":
      • 1,200,000 → "~1.2M ft²"

• Multi-site entries must preserve all sites, separated by semicolons:
  - "51,000 (site); 21,000 (site)" → "~51k (site); ~21k (site)"

• Replace the label **"(estimate)"** with **"(zoominfo)"**.  
• Keep all other tags unchanged:
  - "(site)"  
  - "(public: ZoomInfo 2024)"  
  - "(public: Buzzfile 2024)"  
  - "(LinkedIn 2024)"  
  - "(historical data)"  
  - Any other explicit public-source tag

• Always output exactly one "~" before normalized numeric values.  
• Never duplicate "~" (e.g., do NOT output "~~26k").  

------------------------------------------------------------
2. NUMBER OF EMPLOYEES
------------------------------------------------------------
• Work on the value AFTER it is copied from Raw_Data.  
• If the field is "refer to site" or empty → leave as-is.

• Replace the tag **"(estimate)"** with **"(zoominfo)"**.  
  - "50 (estimate)" → "50 (zoominfo)"  
  - "~40 (estimate)" → "~40 (zoominfo)"

• Do NOT change the numeric portion:
  - Preserve "~", "+", "Over", etc.  
  - Examples:
      • "65+ (site)" → keep exactly as "65+ (site)"  
      • "Over 200 (public: LinkedIn 2024)" → keep exactly

• Keep special tokens and provenance unchanged except for "(estimate)":  
  - "(site)"  
  - "(public: LinkedIn 2024)"  
  - "(public: ZoomInfo 2024)"  
  - "(historical data)"

------------------------------------------------------------
3. ESTIMATED REVENUES
------------------------------------------------------------
• Work on the value AFTER it is copied from Raw_Data.  
• If the field is "refer to site" or empty → leave as-is.

• For any **pure numeric** revenue values (ignoring "$" and commas):  
  - Convert to **"~$XM"** format (use millions; one decimal max when needed).
    • "20000000" → "~$20M"  
    • "5,000,000" → "~$5M"

• Normalize any M-format revenues:
  - "$5M" → "~$5M"  
  - "5M" → "~$5M"  
  - "~$10M" → keep as "~$10M" (already normalized).

• For ranges, maintain the range but normalize each side:
  - "$3–5M" → "~$3–5M"  
  - "4M–6M (estimate)" → "~$4–6M (zoominfo)"

• Replace the tag **"(estimate)"** with **"(zoominfo)"**.  

• Keep all other metadata tags intact:
  - "(public: ZoomInfo 2024)"  
  - "(site)"  
  - "(est) (source)"  
  - "(public: LinkedIn 2024)"  
  - Any other explicit public-source tags

• Always output exactly one "~" before normalized revenue values; never "~~$5M".  
• Preserve semicolon-delimited multi-values and normalize each segment independently.

------------------------------------------------------------
4. GENERAL NORMALIZATION RULES

------------------------------------

C? — Potential Target
------------------------------------
Use C? when:
• Appears relevant but information incomplete  
• Mixed molding + tooling; unclear core business  
• Limited details on ownership, industries, or equipment

------------------------------------
CX — Excluded
------------------------------------
Use CX for:
• High-volume injection molder with incidental tooling only  
• General contract manufacturer with small in-house toolroom  
• Multinational / non-U.S. HQ  
• Commodity plastics, caps/closures, packaging in huge volumes  
• Not a moldmaker / toolmaker at all

------------------------------------
C — Candidate (Neutral)
------------------------------------
Use when:
• Not enough to justify CY  
• Not ambiguous enough for C?  
• Not clearly excluded as CX

Additional rules

• When tooling evidence is extremely thin but not clearly excluded → choose C.
